# Documentação do Projeto

## Fonte canônica

Os arquivos `docs/01-15` são a fonte oficial de verdade do produto, da arquitetura, da segurança e da ordem de implementação.

Ordem recomendada de leitura:

1. `02-regras-negocio.md`
2. `03-arquitetura.md`
3. `10-autenticacao-seguranca.md`
4. `04-banco-dados.md`
5. `05-backend-edge-functions.md`
6. `15-plano-de-implementacao.md`

## Pasta `Specs`

`docs/Specs/` é material auxiliar.

Ela pode apoiar a estruturação de telas, componentes e documentação mais detalhada, mas não substitui as regras de negócio e de segurança definidas nos arquivos `docs/01-15`.

Se existir divergência, prevalecem sempre os documentos canônicos.
